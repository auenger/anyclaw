# 上下文穿梭 (Context Shuttle)

> 一种创新的 LLM 上下文管理策略

## 问题

传统 Agent 每轮对话都要背着全部技能 description，Token 越来越多：

```
Round 1: [17个技能描述: 1700 tokens] + 对话 = 2000 tokens
Round 2: [17个技能描述: 1700 tokens] + 旧对话 + 新对话 = 4000 tokens
Round 3: [17个技能描述: 1700 tokens] + 旧对话 + 旧结果 + 新对话 = 6000 tokens
                          ↑
                     每次都重复加载
```

## 核心思想

**技能选择是"思考过程"，不应该进入历史。**

就像人脑做决策——你会记住"我决定用 git 提交"，但不会记住脑子里闪过的所有选项。

## 穿梭流程

```
                              用户输入
                                 │
                                 ▼
┌────────────────────────────────────────────────────────────────┐
│                                                                │
│   ╔════════════════════════════════════════════════════════╗   │
│   ║  阶段 1: 技能选择 (临时上下文 - 不写历史)               ║   │
│   ╠════════════════════════════════════════════════════════╣   │
│   ║                                                        ║   │
│   ║   Context:                                             ║   │
│   ║   ┌──────────────────────────────────────────────┐    ║   │
│   ║   │ History (不变)                                │    ║   │
│   ║   │ [msg1, msg2, msg3...]                         │    ║   │
│   ║   └──────────────────────────────────────────────┘    ║   │
│   ║   ┌──────────────────────────────────────────────┐    ║   │
│   ║   │ 技能摘要列表 (临时，会被删除)                  │    ║   │
│   ║   │ "commit: 提交代码到 Git 仓库"                 │    ║   │
│   ║   │ "pdf: 处理 PDF 文档..."                       │    ║   │
│   ║   │ "loop: 循环执行任务..."                       │    ║   │

│   ║                         ↓                              ║   │
│   ║                                                        ║   │

│                                 │                             │

│   ╠════════════════════════════════════════════════════════╣   │
│   ║                                                        ║   │
│   ║   Context:                                             ║   │
│   ║   ┌──────────────────────────────────────────────┐    ║   │
│   ║   │ History (不变)                                │    ║   │
│   ║   │ [msg1, msg2, msg3...]                         │    ║   │
│   ║   └──────────────────────────────────────────────┘    ║   │
│   ║   ┌──────────────────────────────────────────────┐    ║   │
│   ║   │ 决策摘要 (精简，进入历史)                     │    ║   │
│   ║   │ "已选: commit, pdf"                          │    ║   │
│   ║   │ "原因: 提交代码并生成 PDF"                    │    ║   │
│   ║   └──────────────────────────────────────────────┘    ║   │
│   ║   ┌──────────────────────────────────────────────┐    ║   │
│   ║   │ 选中技能完整内容 (临时，不进历史)             │    ║   │
│   ║   │ commit: "...用法说明..."                     │    ║   │
│   ║   │ pdf: "...用法说明..."                        │    ║   │
│   ║   └──────────────────────────────────────────────┘    ║   │
│   ║                                                        ║   │
│   ║                         ↓                              ║   │
│   ║                                                        ║   │
│   ║   LLM 执行: git commit + 生成 PDF 文档                 ║   │
│   ║                                                        ║   │
│   ╚════════════════════════════════════════════════════════╝   │
│                                                                │
└────────────────────────────────────────────────────────────────┘
                                 │
                                 ▼
                           返回给用户
```

**Token 节省来源**：
1. 技能列表不进入历史（每轮省 ~1200 tokens）
2. 决策摘要替代完整选择过程（省 ~100 tokens）
3. 选中技能可缓存（后续轮次省更多）

## 关键设计

### 1. 技能摘要列表 (需要 description！)

LLM 需要知道每个技能是干嘛的才能做选择：

```
┌────────────────────────────────────────────────────────────┐
│ <skills>                                                   │
│   <skill name="commit">                                    │
│     提交代码到 Git 仓库                                     │
│   </skill>                                                 │
│   <skill name="pdf">                                       │
│     处理 PDF 文档，支持创建/合并/拆分/提取                   │
│   </skill>                                                 │
│   <skill name="loop">                                      │
│     循环执行任务，支持定时调度                               │
│   </skill>                                                 │
│   <skill name="documents">                                 │
│     文档处理，支持 DOCX/XLSX/PPTX                          │
│   </skill>                                                 │
│   ...                                                      │
│ </skills>                                                  │
│                                                            │
│ 请分析用户需求，选择需要的技能                               │
└────────────────────────────────────────────────────────────┘
          │
          ▼ 约 500 tokens (vs 完整 1700 tokens)
          │
          │  🔮 穿梭后整个列表删除，不进历史！
          ▼
```

**关键**：虽然有 description，但穿梭后会**整个删掉**，不进入历史。

### 2. 不需要 skill.select 工具

LLM 直接用自然语言回答选择：

```
用户: "帮我提交代码并生成 PDF"

LLM 思考后回答:
┌────────────────────────────────────────────────────────────┐
│ 我需要以下技能:                                             │
│                                                            │
│ 1. commit - 因为用户要提交代码到 Git                        │
│ 2. pdf - 因为用户要生成 PDF 文档                            │
│                                                            │
│ 不需要 loop、documents 等其他技能                           │
└────────────────────────────────────────────────────────────┘
```

然后我们解析这个回答，提取出 `["commit", "pdf"]`。

### 3. 穿梭点：删除技能列表，只留决策

```
穿梭前 (选择阶段 - 临时上下文):
┌──────────────────────────────────────────────────────┐
│ History: [msg1, msg2]                                │
│ ──────────────────────────────────────────────────── │
│ System: "<skills>                                    │
│   commit: 提交代码到 Git 仓库                         │  ← 技能列表
│   pdf: 处理 PDF 文档...                              │    (会被删除！)
│   loop: 循环执行任务...                              │
│   documents: 文档处理...                             │
│ </skills>                                            │
│ 请选择需要的技能"                                     │
│ ──────────────────────────────────────────────────── │
│ User: "帮我提交代码并生成 PDF"                        │
└──────────────────────────────────────────────────────┘

                    │ 🔮 穿梭
                    │ 删除整个技能列表
                    │ 只保留决策结果
                    ▼

穿梭后 (执行阶段 - 正式上下文):
┌──────────────────────────────────────────────────────┐
│ History: [msg1, msg2]                                │  ← 不变！
│ ──────────────────────────────────────────────────── │
│ System: "已选择技能: commit, pdf                      │  ← 决策摘要
│         原因: 提交代码并生成 PDF 文档"                 │    (~50 tokens)
│ ──────────────────────────────────────────────────── │
│ System: "## commit 技能详情                          │  ← 只加载选中的
│         ...完整用法说明..."                          │
│                                                      │
│         ## pdf 技能详情                              │
│         ...完整用法说明..."                          │
│ ──────────────────────────────────────────────────── │
│ User: "帮我提交代码并生成 PDF"                        │
└──────────────────────────────────────────────────────┘
```

**核心**：
- 技能列表 = 临时的，穿梭后**删除**
- 决策结果 = 永久的，进入历史
- 选中技能详情 = 临时的，不进历史（但可缓存）

### 4. 会话缓存

```
┌─────────────────────────────────────────────────────────┐
│                    SessionSkillCache                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   Round 1: 选择 commit, pdf                             │
│            └─→ 缓存 {commit: "...", pdf: "..."}         │
│                                                         │
│   Round 2: 选择 commit, documents                       │
│            └─→ commit 命中缓存 ✓                        │
│            └─→ documents 加载并缓存                     │
│            └─→ 缓存 {commit, pdf, documents}            │
│                                                         │
│   Round 3: 选择 pdf                                     │
│            └─→ pdf 命中缓存 ✓                           │
│            └─→ 无需额外加载                             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 伪代码

```python
async def process_with_shuttle(user_input: str) -> str:
    """带穿梭的处理流程"""

    # ═════════════════════════════════════════════
    # 阶段 1: 技能选择 (临时上下文，不写历史)
    # ═════════════════════════════════════════════

    # 构建技能摘要 (name + description)
    skill_summary = loader.build_skill_summary()  # XML 格式

    temp_context = history + [
        system(skill_summary),        # 技能列表 (临时)
        system("请分析用户需求，选择需要的技能"),
        user(user_input)
    ]

    selection = await llm.call(temp_context)
    # "我需要 commit 和 pdf，因为..."

    selected = parse(selection)  # {skills: ["commit", "pdf"], reason: "..."}

    # ═════════════════════════════════════════════
    # 🔮 穿梭点：删除技能列表，只留决策
    # ═════════════════════════════════════════════

    # 加载选中技能的完整内容
    skill_contents = loader.load_skills_for_context(selected.skills)

    # 构建决策摘要 (精简版，进入历史)
    decision = f"已选择: {', '.join(selected.skills)}\n原因: {selected.reason}"

    # ═════════════════════════════════════════════
    # 阶段 2: 正式执行 (结果写历史)
    # ═════════════════════════════════════════════

    # 注意：不包含 skill_summary！已删除
    exec_context = history + [
        system(decision),           # 精简决策 (~50 tokens)
        system(skill_contents),     # 选中技能完整内容
        user(user_input)
    ]

    response = await llm.call(exec_context)

    # 写入历史 (只有 user + response，没有技能列表)
    history.add_user(user_input)
    history.add_assistant(response)

    return response
```

## 总结

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│                    上下文穿梭                           │
│                    ════════════                         │
│                                                         │
│   核心思想:                                              │
│   ┌───────────────────────────────────────────────┐    │
│   │ 技能选择 = 思考过程 → 不进历史                  │    │
│   │ 技能列表 = 临时的 → 穿梭后删除                 │    │
│   │ 只保留决策结果 → 精简上下文                    │    │
│   └───────────────────────────────────────────────┘    │
│                                                         │
│   效果:                                                  │
│   ┌───────────────────────────────────────────────┐    │
│   │ Token 节省: ~50% (每轮省掉技能列表累积)         │    │
│   │ 前缀缓存: History 不变，命中率高               │    │
│   │ 历史干净: 只有有意义的内容                      │    │
│   └───────────────────────────────────────────────┘    │
│                                                         │
│   代价:                                                  │
│   ┌───────────────────────────────────────────────┐    │
│   │ 每轮需要两阶段 (选择 + 执行)                   │    │
│   │ 但选择阶段 token 少，且有缓存                  │    │
│   └───────────────────────────────────────────────┘    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```
