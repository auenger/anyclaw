# 特性管理器

主代理，管理和编排多个特性，处理依赖关系，优化整体开发工作流。

## 参数

- `--status`: 显示整体工作流状态
- `--optimize`: 建议最佳特性开发顺序
- `--check-dependencies`: 检查所有特性依赖
- `--auto-schedule`: 自动启动就绪的特性
- `--report`: 生成工作流报告

## 执行模式

### 模式 1：状态检查

显示整体工作流状态：
```
/feature-manager --status
```

显示：
- 按状态的当前特性
- 并行使用
- 阻塞和依赖
- 建议

### 模式 2：依赖检查

检查所有特性依赖：
```
/feature-manager --check-dependencies
```

显示：
- 依赖图
- 循环依赖（错误）
- 缺失依赖
- 已满足依赖

### 模式 3：优化

建议最佳开发顺序：
```
/feature-manager --optimize
```

分析：
- 优先级 vs 依赖
- 并行开发机会
- 关键路径
- 瓶颈

### 模式 4：自动调度

自动启动就绪的特性：
```
/feature-manager --auto-schedule
```

启动满足以下条件的特性：
- 没有未满足的依赖
- 在并行限制内
- 具有最高优先级

### 模式 5：生成报告

生成综合工作流报告：
```
/feature-manager --report
```

包括：
- 特性统计
- 时间指标
- 阻塞分析
- 建议

## 示例

