# 实现特性代码

读取规范、分析任务、在 worktree 中实现代码。按照以下步骤执行：

## 参数

- `feature-id`: 特性 ID（必需）
- `--task=<index>`: 只实现特定任务（可选）
- `--dry-run`: 仅分析，不实现（可选）

## 第 1 步：预检查

- 特性必须在 `queue.yaml` 的 active 列表中
- Worktree 必须存在

## 第 2 步：读取特性文档

读取 `features/active-{id}/spec.md`：
- 特性描述
- 上下文分析（参考代码、相关文档）
- 验收标准

读取 `features/active-{id}/task.md`：
- 任务列表
- 任务状态（completed/pending）

## 第 3 步：分析任务

解析任务列表：
1. 识别任务依赖关系
2. 确定实现顺序
3. 列出需要的参考代码/文档

输出分析结果并请求确认。

## 第 4 步：实现代码

切换到 worktree 目录：
```bash
cd {worktree_path}
```

对于每个任务：
1. 实现代码
2. 更新 `task.md` 状态（勾选复选框）
3. 添加简短的实现说明

**实现指南：**
- 参考现有代码风格
- 复用现有组件和工具
- 遵循项目目录结构
- 添加必要的错误处理
- 添加必要的类型定义

## 第 5 步：自测

- 如果有现有测试，运行测试
- 手动验证核心功能
- 检查代码质量
- 记录测试结果

## 输出格式

```
✅ 实现完成

完成任务: 4/4
修改文件: 5

测试: 12 通过, 0 失败

下一步:
  - 运行 /verify-feature {id} 进行验证
  - 或运行 /complete-feature {id} 完成
```

## 注意事项

1. **不自动提交** - 代码保留在 worktree 中，不提交
2. **参考现有代码** - 优先参考 spec 中提到的上下文
3. **保持文档同步** - 实现过程中更新 task.md 状态
4. **支持恢复** - 可以中断后继续
