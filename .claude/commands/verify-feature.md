# 验证特性

验证已实现的特性是否满足所有要求。按照以下步骤执行：

## 参数

- `feature-id`: 要验证的特性 ID（必需）

## 第 1 步：加载特性上下文

读取特性文档：
- `features/active-{id}/spec.md` - 验收标准
- `features/active-{id}/task.md` - 任务列表
- `features/active-{id}/checklist.md` - 完成检查清单

## 第 2 步：检查任务完成

验证 `task.md` 中的所有任务已完成：
- 检查每个复选框
- 验证任务说明
- 识别未完成的任务

## 第 3 步：验证验收标准

审查 `spec.md` 中的 Gherkin 场景：
- 解析每个场景
- 验证实现是否满足标准
- 如可能，测试场景

## 第 4 步：运行测试

执行可用的测试：
- 单元测试
- 集成测试
- UI 特性的手动测试

## 第 5 步：代码质量检查

审查：
- 代码风格合规
- 错误处理
- 边缘情况
- 文档

## 第 6 步：生成报告

输出验证结果：
- 完成状态
- 测试结果
- 发现的问题（如有）
- 建议

## 输出格式

```
📋 特性验证报告: {id}

完成状态:
  任务: ✅ 5/5 完成
  测试: ✅ 12/12 通过
  检查清单: ✅ 所有项已检查

验收标准:
  ✅ 用户注册正常
  ✅ 用户登录正常
  ✅ 用户登出正常
  ⚠️ 权限追踪不完整

发现问题:
  - 权限追踪未完全实现

建议:
  - 在完成特性前完成权限追踪

总体: ⚠️ 发现轻微问题

选项:
  [1] 修复问题并重新验证
  [2] 继续完成特性（将记录问题）
  [3] 退出并手动修复
```

## 退出码

| 码 | 含义 |
|----|------|
| 0 | 验证通过 |
| 1 | 发现轻微问题 |
| 2 | 发现重大问题 |
| 3 | 严重失败 |
