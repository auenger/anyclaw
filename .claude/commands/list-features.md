# 列出特性

列出工作流中的所有特性及其状态、优先级和其他详情。支持过滤和排序。

## 参数

- `--status=<status>`: 按状态过滤（pending, active, blocked, done）
- `--sort=<field>`: 按字段排序（priority, name, created, size）
- `--format=<format>`: 输出格式（table, json, detailed）

## 第 1 步：加载队列数据

读取 `feature-workflow/queue.yaml`：
- pending 列表
- active 列表
- blocked 列表

读取 `feature-workflow/archive-log.yaml`：
- 已完成的特性

## 第 2 步：应用过滤

如果提供了状态过滤：
- `--status=pending`: 只显示待处理特性
- `--status=active`: 只显示活跃特性
- `--status=blocked`: 只显示被阻止的特性
- `--status=done`: 只显示已完成的特性

## 第 3 步：排序结果

按指定字段排序（默认：priority）：
- `priority`: 按优先级排序（高到低）
- `name`: 按字母顺序排序
- `created`: 按创建日期排序
- `size`: 按大小排序（S, M, L）

## 第 4 步：格式化输出

根据 --format 参数格式化结果：
- `table`: ASCII 表格格式
- `json`: JSON 格式
- `detailed`: 每个特性的完整详情

## 详细格式

显示每个特性的完整详情，包括：
- 基本信息（id, name, priority, size）
- 依赖关系
- 父/子关系
- 创建日期
- 对于活跃特性：分支、worktree、进度
- 对于被阻止的特性：阻止原因
