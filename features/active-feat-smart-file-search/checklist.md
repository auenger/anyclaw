# Smart File Search - 完成检查清单

## 功能验收

### VP1: 智能搜索策略
- [ ] 按优先级顺序搜索（Downloads > Desktop > Documents > Projects > Home）
- [ ] 文件类型关联目录（.xlsx → Downloads）
- [ ] 找到第一个匹配即返回
- [ ] 排除缓存目录（.git, node_modules, Library）

### VP1.5: 信息不足主动询问
- [ ] 无文件名时主动询问
- [ ] 模式太宽泛时建议限定目录
- [ ] 有足够信息时直接搜索
- [ ] 返回结构化询问建议

### VP1.6: 对话上下文关联
- [ ] 从对话历史提取路径
- [ ] 按新鲜度排序上下文路径
- [ ] 过滤过期路径（>10 轮对话）
- [ ] 合并到搜索优先级

### VP2: 搜索缓存
- [ ] 缓存成功找到的文件位置
- [ ] LRU 淘汰策略（max=100）
- [ ] 常用目录缓存（访问 > 3 次）
- [ ] 24 小时过期清理
- [ ] 缓存持久化（启动加载/关闭保存）

### VP3: 动态路径授权
- [ ] 访问受限目录时抛出 `AuthorizationRequiredError`
- [ ] 危险路径直接拒绝，不提示授权（~/.ssh, /etc/passwd）
- [ ] 临时授权（仅当前会话）
- [ ] 持久化授权（写入配置文件）
- [ ] 授权状态全局共享（单例模式）

### VP3.5: 跨渠道授权交互
- [ ] `BaseChannel.request_authorization()` 抽象方法
- [ ] CLI 渠道：rich.prompt 交互式菜单
- [ ] Discord 渠道：Button 组件授权
- [ ] 飞书渠道：交互式卡片授权
- [ ] 授权超时处理（60s 超时视为拒绝）
- [ ] AgentLoop 集成授权流程

### VP4: 搜索工具增强
- [ ] `search_files` 工具可用
- [ ] 支持 pattern, search_paths, file_type, max_depth, use_cache 参数
- [ ] 格式化输出（搜索用时、缓存命中数）
- [ ] 超时控制（默认 5 秒）

## 代码质量

- [ ] 所有新代码有类型注解
- [ ] 使用 Black 格式化
- [ ] Ruff 检查通过
- [ ] 无安全漏洞（危险路径检查）
