# 特性规范：迭代限制智能汇报

## 概述

**ID**: feat-iteration-summary
**名称**: 迭代限制智能汇报
**优先级**: 70
**大小**: M

## 背景

当 Agent 达到最大迭代次数时，当前实现只是简单返回 `"达到最大迭代次数"`，用户体验差：
- 用户不知道执行了什么操作
- 不知道处理到什么阶段
- 不知道可能卡在哪里
- 没有下一步建议

## 用户价值点

### 价值点 1: 工具调用摘要与进度汇报

当达到最大迭代次数时，系统自动生成包含以下内容的智能汇报：

- **已执行工具摘要**: 按时间序列列出所有工具调用
- **执行状态统计**: 成功/失败数量、耗时统计
- **当前处理阶段**: 分析任务执行到哪一步
- **卡点诊断**: 识别可能的问题点（重复调用、错误循环等）
- **下一步建议**: 给用户具体的后续操作建议

## 验收场景 (Gherkin)

### 场景 1: 正常达到迭代限制时生成汇报

```gherkin
Feature: 迭代限制智能汇报

Scenario: 达到最大迭代次数时生成智能汇报
  Given Agent 正在处理用户请求
  And 已执行 10 次迭代（默认最大值）
  And 执行了多个工具调用
  When 达到迭代限制
  Then 系统调用 LLM 分析整个对话历史
  And 返回包含以下内容的工作进度汇报:
    | 内容类型 | 说明 |
    | 工具摘要 | 列出所有已执行的工具及结果状态 |
    | 进度分析 | 描述当前执行到什么阶段 |
    | 卡点诊断 | 分析可能卡住的原因 |
    | 下一步建议 | 给出用户可操作的后续步骤 |
```

### 场景 2: 无工具调用时达到限制

```gherkin
Scenario: 无工具调用时达到迭代限制
  Given Agent 正在处理用户请求
  And 迭代过程中没有执行任何工具
  When 达到迭代限制
  Then 返回简洁的进度汇报
  And 说明可能需要增加迭代次数或简化任务
```

### 场景 3: 错误循环检测

```gherkin
Scenario: 检测到重复工具调用循环
  Given Agent 正在处理用户请求
  And 同一个工具以相同参数被调用超过 3 次
  When 达到迭代限制
  Then 汇报中明确指出检测到循环
  And 分析可能的循环原因
  And 给出打破循环的建议
```

### 场景 4: SubAgent 迭代限制汇报

```gherkin
Scenario: SubAgent 达到迭代限制
  Given SubAgent 正在后台执行任务
  And 达到 subagent_max_iterations (默认 15)
  When 生成汇报
  Then 汇报被正确传递给父 Agent
  And 用户能看到完整的子任务执行摘要
```

## 技术设计

### 汇报生成流程

```
达到迭代限制
    ↓
收集迭代数据
├── 工具调用记录（名称、参数、结果摘要、耗时）
├── 消息历史（用户意图、Agent 响应）
└── 状态统计（成功/失败/重复调用）
    ↓
构建分析 Prompt
├── 注入工作进度分析指令
├── 提供工具调用时间线
└── 要求结构化输出
    ↓
调用 LLM 生成汇报
    ↓
返回用户友好的进度汇报
```

### 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `agent/loop.py` | 修改 `_run_with_tools()` 返回值逻辑 |
| `agent/tool_loop.py` | 修改 `ToolCallingLoop` 返回值逻辑 |
| `agent/summary.py` | 新增：迭代摘要生成器 |
| `agent/subagent.py` | 适配 SubAgent 的汇报逻辑 |
| `config/settings.py` | 新增 `iteration_summary_enabled` 配置项 |

### 配置项

```toml
[agent]
# 达到迭代限制时是否生成智能汇报（默认启用）
iteration_summary_enabled = true
# 汇报生成的最大 token 数
iteration_summary_max_tokens = 1000
```

## 非目标

- 不改变迭代限制的触发机制
- 不增加额外的 LLM 调用（只在达到限制时调用一次）
- 不修改工具调用循环的核心逻辑

## 成功指标

1. 用户在达到迭代限制时能看到有意义的进度信息
2. 汇报内容能帮助用户理解当前状态
3. 汇报中的建议对用户有实际帮助
